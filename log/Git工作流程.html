两种工作流程

协作开发工作流程 : 这种情况是最复杂的情况, 多人团队共同开发一个项目;
-- 与远程仓库同步 : git pull ;
-- 修改文件 : 添加 删除 修改文件;
-- 查看变更 : git status ;
-- 载入变更 :添加文件, 先使用git add fileName, 在使用 git commit -m 'note' 载入变更; 如果是删除 修改文件, 直接使用 git commit -m 'note' 提交;
-- 重复 : 重复执行 修改文件 查看变更 载入变更 提交载入动作;
-- 上传 : 使用 git push 命令将项目源码提交带GitHub中去;
